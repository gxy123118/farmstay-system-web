# Farmstay System Web V2 前端路线图

## 1. 目标
- 在保留现有 Vue 3 项目的基础上，完成游客端与经营者端的前端升级。
- 让首页承担“浏览 + 搜索 + AI 助手入口”的统一入口职责。
- 让个人中心聚焦“订单、店铺资料、经营建议”等核心工作台能力。

## 2. 当前基线
- 已有页面：首页、登录、农家乐详情、支付页、订单详情、评价页、个人中心。
- 已有能力：基础搜索、下单、支付跳转、订单管理、经营者基础管理。
- 主要问题：
1. 前端文档为英文，不利于后续协作。
2. AI 助手放在个人中心中，入口层级不合理。
3. 经营者“店铺资料管理”布局松散，信息与操作混杂。
4. 首页之前缺少统一分页和悬浮式智能交互入口。

## 3. V2 前端方向
- 首页：
1. 承担搜索、推荐、分页浏览。
2. 提供可拖拽、可展开的悬浮 AI 小助手。
- 个人中心：
1. 游客聚焦订单与评价。
2. 经营者聚焦订单、店铺资料、经营建议。
- 经营者端：
1. 店铺管理采用“左侧列表 + 右侧编辑区”的工作台结构。
2. 后续继续承接房型、活动、餐饮、优惠券等扩展能力。

## 4. 本轮实施内容
- 已完成：
1. 前端产品文档改为中文。
2. 首页支持农家乐分页浏览。
3. AI 助手改为首页悬浮入口，可拖拽、可展开、支持对话模式。
4. 个人中心移除 AI Tab。
5. 经营者“店铺资料管理”重做为工作台布局。
- 暂未完成：
1. AI 流式输出。
2. 店铺管理继续拆分出房型/餐饮/活动子模块。
3. 支付正式链路完成后的前端支付改版。

## 5. 后续阶段
## 第一阶段
- 完善经营者工作台的信息层级。
- 将更多经营能力统一收口到店铺工作台内。

## 第二阶段
- 接入真实 Spring AI 能力。
- 增强 AI 助手的上下文感知、会话保持和推荐能力。

## 第三阶段
- 支付链路标准化后，重做支付页与状态流转体验。
- 补齐经营统计与财务视图。

## 6. 工程要求
- 保持 Vue 3 + Vite + TypeScript。
- 公共接口统一走 `src/services/api.ts`。
- 新增复杂交互优先抽为组件，不在页面内堆积逻辑。
- 保持当前乡野、编辑感的视觉方向，但工作台区域要更清晰、更可操作。
